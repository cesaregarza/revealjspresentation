Goals

  • Create a stable, robust back-end
  • Create a user-friendly front-end
  • Create a secure but flexible database

Technologies

  • MEAN Stack
  • MongoDB
  • Express
  • Angular 6
  • Node.js
  • RESTful API
  • Cross-Origin Resource Sharing (CORS)

Serverside (Back-End)

We wanted to have a powerful but flexible back-end that we could keep adding to in the future and new people can come along and edit it easily

Node.js

  • Powerful
  • Flexible
  • Most Popular Framework (According to StackOverflow)

Express

  • Framework for node.js
  • Easy to learn
  • Widespread developer support
  • Sublime documentation

Authentication

Passport.js

  • Very flexible, tons of "strategies" available online
  • Powerful
  • Injects itself into express-sessions
  • Simple to use for token validation
  • Awful, outdated documentation

Security

  • Security is very important to us
  • Passwords hashed using state-of-the-art encryption in scrypt
  • HTTPS fully implemented throughout the webapp
  • Authentication guards placed at crucial areas
  • Server currently only accepts valid requests from the front-end's URL.

Clientside (Front-End)

We wanted to have a user-friendly, single-page application that was flexible but powerful enough to handle anything we throw at it

Angular 6

  • We have experience with it!
  • Modular
  • Powerful CLI tool
  • Subpar documentation Excellent community support

SASS

  • Extremely easy to use
  • Angular already supports it
  • Is a superset of CSS

Database

MongoDB

  • Mongoose is very easy to learn
  • Dynamic
  • Very very fast

Deleted or Delayed Features

  • Liking animals and keeping tabs on them
  • Public use API
  • GPS-based search limiting
  • Robust options

Conclusion

The project was deceptively difficult, with authorization and persistent login sessions taking up about half of our time. The time constraints imposed upon us were probably the most stressful aspect of development, this experience has taught us how time management is one of the more difficult and most easily underestimated parts of development.
